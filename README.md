# Predicting Student Health Risk (Kaggle)

Классификация здоровья студентов на три класса (fit / at-risk / unhealthy) по ~300k строк: сон, активность, питание, стресс. Сильный дисбаланс — at-risk это 86% выборки. Метрика — balanced accuracy.

LightGBM с `class_weight='balanced'`, early stopping по валидационному сплиту. Пропуски заполняются средними по группам (шаги — по уровню активности, длительность сна — по качеству сна), статистики считаются на трейне и применяются к тесту. Категории закодированы порядково.

Итог: **0.86** на лидерборде.

```bash
python model.py
```

Ожидает train.csv и test.csv от соревнования, пишет submission.csv.

## Путь и ошибки

0.64 -> 0.849 -> 0.86. Первый скачок — вернул признаки, которые сам же выкинул: посчитал VIF, увидел мультиколлинеарность и удалил шесть непрерывных фичей. VIF — диагностика для линейных моделей, бустингу коррелированные признаки не мешают. Это удаление стоило ~0.2 скора.

Второй скачок — прочитал наконец, что метрика соревнования balanced accuracy, а не accuracy. До этого видел 0.94 accuracy на валидации при 0.849 на лидерборде: при дисбалансе 86/8/6 обычная accuracy почти целиком описывает мажоритарный класс. После перехода на правильную метрику локально + class_weight='balanced' валидация и лидерборд наконец стали двигаться вместе.

Отдельный урок: без локальной валидации оба косяка были невидимы. Первая версия обучалась и сразу сабмитилась — сравнивать конфиги можно было только сабмитами.

## Что не пробовал

Сырые NaN и category dtype вместо ручной импутации и кодирования (LightGBM умеет и то и другое нативно), CatBoost, подбор порогов по predict_proba. У лидеров 0.93+, разрыв скорее всего где-то здесь.
